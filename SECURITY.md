# Security Policy

This repo is mostly docs, skills, and example artifacts, but security still matters because people may copy these patterns into production systems.

## Reporting
Please report security-sensitive issues via [GitHub Security Advisories](https://github.com/xuanhieu2611/build-your-own-agents-skill/security/advisories/new) before opening a public issue.

## Expectations
Prefer:
- least-privilege defaults
- approval for high-impact actions
- safe retries
- auditable logging guidance
